Each commands in the factor analysis tutorials will be generated once per theme. The MWE is considered a one theme analysis

Merge covariate#

This step generate a concatanated principle component + covaraiate (pc-cov) matrixs so that we can use it to generate a residual phenotype, as outlined in the phenotype processing page

The tolerance for sample-wide NA rate of any covariates/pc are specified by tol_cov. -1 means quit, otherwise for covariate with missing rate larger than tol_cov will be removed, with missing rate smaller than tol_cov will be mean_imputed.

The first k PCs that satisfied >70% PVE will be merged with the covariates for downstream analysis and for estimating the residual expression in the phenotype_preprocessing section.


 sos run pipeline/covariate_formatting.ipynb merge_pca_covariate \
        --cwd output/data_preprocessing/protocol-eqtl/covariates \
        --pcaFile output/data_preprocessing/protocol-eqtl/pca/xqtl_protocol_data.merged.protocol-eqtl.related.filtered.extracted.pca.projected.rds \
        --covFile  ptotocol_data.covariate.tsv \
        --tol_cov 0.3  \
        --k `awk '$3 < 0.7' output/data_preprocessing/protocol-eqtl/pca/xqtl_protocol_data.merged.protocol-eqtl.related.filtered.extracted.pca.projected.scree.txt | tail -1 | cut -f 1 ` \
        --container containers/bioinfo.sif  --name ptotocol_data.covariate.pca --outliersFile output/data_preprocessing/protocol-eqtl/pca/xqtl_protocol_data.merged.protocol-eqtl.related.filtered.extracted.pca.projected.outliers

Optional: Removal of outliers (Not part of the protocol data analysis)#

At this step, based on the output plot of PCA step, the user can decide if outliers should be removed. By default, the outlierFile will be filled, but the user have to manually add the --remove-outliers flag which will remove all the samples ID in the second column of the .outlier file from the header of pc-cov matrix, effectively remove them from all down stream analysis.

sos run pipeline/covariate_formatting.ipynb merge_pca_covariate \
        --cwd output/data_preprocessing/MWE/covariates \
        --pcaFile data_preprocessing/MWE/pca/MWE.MWE.related.filtered.extracted.pca.projected.rds \
        --covFile  MWE.covariate.cov.gz \
        --tol_cov 0.3  \
        --k `awk '$3 < 0.7' output/data_preprocessing/MWE/pca/MWE.MWE.related.filtered.extracted.pca.projected.scree.txt | tail -1 | cut -f 1 ` \
        --container containers/bioinfo.sif  --outliersFile data_preprocessing/MWE/pca/MWE.MWE.related.filtered.extracted.pca.projected.outliers --remove-outliers

Residual Expression#

The residual expression will be computed using the covariate-PCs matrixs. Residual expression is needed because PEER(MOFA2) handle covariates poorly. However, the residual expression is only used for factor analysis.

  sos run pipeline/covariate_formatting.ipynb compute_residual \
        --cwd output/data_preprocessing/protocol-eqtl/resid_phenotype \
        --phenoFile output/rnaseq/xqtl_protocol_data.rnaseqc.low_expression_filtered.outlier_removed.tmm.expression.bed.gz \
        --covFile  output/data_preprocessing/protocol-eqtl/covariates/ptotocol_data.covariate.pca.gz \
        --container containers/bioinfo.sif  --name xqtl_protocol_data.rnaseqc.low_expression_filtered.outlier_removed.tmm.expression.cov_pca




  sos run pipeline/covariate_formatting.ipynb compute_residual \

Factor Analysis#

The residual expression will be used to conduct the factor analysis using either BiCV (APEX) or PEER (MOFA2). The purpose of factor analysis is to uncovered un-measured factor embedded in the phenotype data, potentiall factor could be experiment batch effct, or unmeasure morbidity of the sampels .etc.

The primary factor analysis method we used in our analysis is PEER.

 sos run pipeline/PEER_factor.ipynb PEER \
   --cwd output/data_preprocessing/protocol-eqtl/covariates \
   --phenoFile output/data_preprocessing/protocol-eqtl/resid_phenotype/xqtl_protocol_data.rnaseqc.low_expression_filtered.outlier_removed.tmm.expression.cov_pca.resid.bed.gz  \
   --container containers/PEER.sif  \
   --walltime 24h \
   --numThreads 8 \
   --iteration 1000 \
   --N 0  




 sos run pipeline/PEER_factor.ipynb PEER \
   --cwd output/data_preprocessing/protocol-sqtl/covariates \
   --phenoFile output/data_preprocessing/protocol-sqtl/resid_phenotype/xqtl_protocol_data_bam_list_intron_usage_perind.counts.gz_raw_data.qqnorm.formated.cov_pca.resid.bed.gz  \
   --container containers/PEER.sif  \
